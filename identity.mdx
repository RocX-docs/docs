# Your activity defines you

A wallet can identify an account.

It cannot explain the person behind it.

In Web3, anyone can create a new wallet. Trust, however, has to be built through time and activity.

RocX Identity is created from that history.

> Wallets identify accounts.
>
> Activity identifies people.

---

## What RocX Identity is

RocX Identity is an on-chain profile built from participation, contribution, reputation, and trust.

It grows as a user interacts with the ecosystem, verifies activity through Proof of Activity, and builds reputation over time.

<Info>
RocX Identity connects what a user does, how they participate, and what trust they earn into one on-chain profile.
</Info>

## Why identity matters

Most Web3 systems remember wallets.

RocX is designed to remember more context: how long a user participated, what they contributed, and what kind of trust they built.

That context matters because two wallets can look similar while the people behind them have very different histories.

## Wallet and RocX Identity

| Wallet | RocX Identity |
| --- | --- |
| Address | Explorer Passport |
| Assets | Activity |
| Transactions | Reputation |
| Ownership | Contribution |
| Private key | Proof of Activity |

<Tip>
RocX Identity does not ask users to reveal more personal information. It builds identity from participation that already happens inside the ecosystem.
</Tip>

## Explorer Passport

Explorer Passport brings RocX Identity into one profile.

It collects the signals that explain how a user participates and how that participation grows over time.

## The RocX perspective

RocX sees identity as a record built over time.

It should not depend only on documents, account ownership, or one-time verification. It should also reflect what a user has done and what trust they have earned.

<Note>
Identity is not assigned once.

It is accumulated through activity.
</Note>
